# Contributing

Thank you for improving AI Idea Hunter. Read the product requirements, architecture, project rules, and active tasks before changing code.

## Workflow

1. Search existing issues and pull requests.
2. Use an issue to define the problem, scope, non-goals, and acceptance criteria.
3. Create a short-lived branch from `main`.
4. Implement one focused outcome with tests and documentation.
5. Run `npm run test:all` and manually check affected browser behavior.
6. Open a pull request with evidence, risks, compatibility impact, and rollback.
7. Resolve review threads and wait for required checks.

Do not submit secrets, personal data, unsafe third-party assets, unverifiable claims, or vulnerability details. Report security issues according to [SECURITY.md](SECURITY.md).

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
